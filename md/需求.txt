🎯 任务目标
通过本编码挑战，你将学习并展示：
👉 如何构建类似 OpenAI Operator 的生产级 AI Agent 前端系统

⏰ 时间要求
截止时间：24 小时 
实际建议投入：2–4 小时 
允许使用 AI 编程工具（Claude Code / Gemini CLI / Codex 等） 

🧩 任务要求
1️⃣ 基于现有 computer use 项目扩展
起点项目：
👉 https://github.com/vercel-labs/ai-sdk-computer-use
你需要在这个 demo 基础上升级成一个“生产级 AI Agent Dashboard”。

2️⃣ 构建一个双栏 Dashboard UI
必须实现左右两栏布局：
左侧（Chat 区域）
AI 聊天窗口 
tool call（工具调用）需要内联展示 
一个可折叠的 debug 面板 
右侧（执行区）
VNC viewer（远程桌面画面） 
tool call 详细信息展示区 
额外要求：
左右面板可以水平拖拽调整宽度 

3️⃣ 事件流（Event Pipeline）+ 类型状态管理
你需要设计一个“事件系统”，包含：
必须记录：
每一次 tool call：
id 
timestamp（时间） 
type（工具类型） 
payload（参数） 
status（状态） 
duration（耗时） 

派生状态（Derived State）：
各类型 tool call 的统计数量 
agent 当前状态（idle / running / error 等） 

Debug 面板：
可视化整个 event store 
用来调试 agent 行为流 

4️⃣ 多会话 Chat History
必须支持：
创建 session 
切换 session 
删除 session 
chat history 本地持久化（localStorage） 

5️⃣ TypeScript 强约束
必须遵守：
❌ 禁止 any 
✅ 使用 discriminated unions（可辨别联合类型） 
类型系统要清晰、可扩展 

6️⃣ 性能要求（重点）
👉 VNC viewer 不能因为 chat 更新而重新渲染
（说明你要做 memo / state isolation / selector 优化）

⭐ Bonus
移动端适配（responsive layout）
